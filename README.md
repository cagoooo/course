# 課表排課系統 (STC) 檔案分析與使用手冊

## 📂 1. 資料夾內容總覽
經過分析，此資料夾存放了一套 Windows 桌面版的學校排課系統 (STC) 及其相關資料庫。

### 核心程式
*   **`STC.EXE`**: 排課系統的主執行檔。這是一個傳統的 Windows 應用程式 (Win32)，負責執行排課演算法與介面操作。
*   **`Stc.INI`**: 系統初始化設定檔。
    *   內容包含：學校名稱 (如「石門國民小學」)、作業學期 (如 `111-1`)、列印設定 (紙張大小 A3/B4)、備份與匯出路徑等。

### 資料目錄 (例如 `Ys110-1`, `Ys111-1`)
這些資料夾代表不同學年/學期的排課資料 (e.g., `Ys110-1` 代表 110 學年度第 1 學期)。內部檔案通常為純文字格式 (Big5 編碼) 或專有格式：
*   **`TeachNam`**: 教師名單資料庫。
*   **`CoursNam`**: 科目/課程名稱資料庫。
*   **`ClassTab`**: 班級課表結構資料 (由數字代碼組成的矩陣)。
*   **`RoomNam`**: 教室名稱資料。
*   **`ClassCur`**, **`ClassNum`**: 推測為班級課程設定或代碼對照表。

## 🚀 2. 使用與操作說明 (本地端)

由於這是一個 `.EXE` 執行檔，**必須在 Windows 電腦上執行**。

1.  **啟動**: 雙擊資料夾中的 `STC.EXE`。
2.  **載入資料**: 程式啟動後，通常會讀取 `Stc.INI` 中的 `作業期別` (如 `111-1`) 來自動載入對應資料夾 (`Ys111-1`) 的資料。
    *   若要切換學期，可能需要在程式內設定，或手動修改 `Stc.INI` 中的 `作業期別` 數值。
3.  **匯出功能**:
    *   根據 `Stc.INI` 的設定 (`匯出磁碟=d:\`)，程式應具備將課表資料匯出到指定路徑的功能。
    *   建議檢查軟體介面中是否有「匯出網頁」、「匯出 Excel」或「列印 HTML」的選項。

## ☁️ 3. GitHub 移植與部署可行性分析

您詢問「是否能完整移植到 GitHub 做部署使用」，以下是專業評估：

### ❌ 無法直接部署主程式 (The App Itself)
*   **原因**: GitHub (包含 GitHub Pages) 是一個網頁託管服務，主要支援 HTML/JavaScript/CSS 網頁程式。它**無法執行** Windows 的 `.EXE` 檔案。
*   **結論**: 您**不能**把這個資料夾推上 GitHub Pages 然後期待使用者打開網頁就能操作這個排課軟體。

### ✅ 可部署靜態成果 (The Output)
*   **方案**: 如果您的目標是在網頁上「顯示課表」，而非「操作排課」：
    1.  在本地端打開 `STC.EXE`。
    2.  尋找「匯出 HTML」或「列印成網頁」功能。
    3.  將匯出後的 `.html` 檔案上傳到 GitHub。
    4.  啟用 GitHub Pages，即可讓師生透過網頁查詢課表。

### ⚠️ 進階移植方案 (需程式開發)
*   如果一定要在網頁上動態顯示這些原始資料 (`ClassTab`, `TeachNam` 等)，您需要開發一個網頁解析器 (Web Parser)：
    *   **難度**: 高。需自行研究 `ClassTab` 的數據結構，並用 JavaScript 寫程式將其轉譯為課表表格。
    *   **編碼問題**: 原始檔案使用 Big5 編碼，現今網頁 (UTF-8) 需做轉碼處理才能正常顯示中文。

## 💡 總結建議
1.  此資料夾可上傳 GitHub 作為**備份 (Backup)**。
2.  若要**部署使用**，請先在本地匯出 HTML 課表，再將 HTML 上傳至 GitHub Pages。
